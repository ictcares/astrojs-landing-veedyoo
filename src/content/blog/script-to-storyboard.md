Many creators write a full script first and then ask, "What should be on screen?" That creates a problem: the visuals become decoration instead of part of the argument.

A better workflow is to split the script into scenes while you write.

## Start with scene jobs

Every scene should have a job. It might introduce a question, explain a mechanism, show evidence, compare two options, or close a chapter.

If a scene has no job, it usually becomes filler.

## Match narration length to visual capacity

A dense visual can hold attention longer than a simple title card. A static quote cannot carry a long paragraph. Match duration to what the viewer can actually inspect.

Useful checks:

- Can the viewer understand the visual in the time given?
- Does the narration repeat what the visual already says?
- Does the scene need motion, a cut, or a different layout?

## Write visual prompts as production notes

A storyboard prompt should not only describe an image. It should explain the intent of the shot.

Better prompt structure:

1. Subject
2. Composition
3. Visual style
4. Scene purpose
5. Continuity rule

That gives the scene more direction than a loose image request.

## Keep continuity visible

A long video needs recurring rules: colors, typography, camera movement, visual motifs, and things to avoid. Keep those notes close to the storyboard so each scene follows the same creative language.

This is the difference between a collection of generated frames and a video that feels planned.
